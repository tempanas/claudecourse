# Consultation Request Triage Skill

Учебный проект для продвинутого варианта домашнего задания CLs03, урок 6 «Скиллы в Claude Code». Проект моделирует безопасный разбор обезличенных заявок студентов на консультацию.

## Созданный skill

Проектный skill `consultation-request-triage`:

- проверяет полноту заявки;
- определяет группу тарифа;
- предварительно классифицирует консультацию как бесплатную, платную или требующую уточнения;
- выявляет риски по вопросам, оплате, слоту, Zoom и фиксации в таблице;
- формирует следующие внутренние действия для должности Зерокодер19;
- сохраняет структурированный результат в `reports/2026-06-consultation-request-triage-result.md`.

Инструкция skill находится в:

`.claude/skills/consultation-request-triage/skill.md`

## Как тестировать

1. Открыть проект в Claude Code.
2. Убедиться, что проектная инструкция из `CLAUDE.md` загружена.
3. Выбрать одну заявку из `data/demo-consultation-request.md`.
4. Попросить использовать skill `consultation-request-triage` для разбора выбранной заявки.
5. Проверить, что ответ соответствует JSON-схеме из `skill.md`.
6. Проверить файл `reports/2026-06-consultation-request-triage-result.md`.
7. Убедиться, что в отчёте нет реальных персональных данных, контактов, внешних ссылок и утверждений о выполненных внешних действиях.

Пример тестового запроса:

```text
Используй skill consultation-request-triage.
Разбери Заявку 2 из data/demo-consultation-request.md
и сохрани результат в reports/2026-06-consultation-request-triage-result.md.
Никаких внешних действий не выполняй.
```

## Что приложить к домашнему заданию

- ссылку на репозиторий или архив проекта;
- файл `.claude/skills/consultation-request-triage/skill.md`;
- `CLAUDE.md`;
- `docs/consultation-reglament-summary.md`;
- `data/demo-consultation-request.md`;
- пример сформированного отчёта из `reports/`;
- скриншот запуска skill и итогового результата в Claude Code;
- краткое описание рабочей рутины и правил безопасности.

Исходный PDF-регламент остаётся в локальной рабочей папке, но не должен включаться в публичный репозиторий или архив домашнего задания, так как может содержать внутренние рабочие ссылки. Папка `tmp/` также не включается в сдачу. В учебных Markdown-файлах используются только обезличенные выводы без реальных ссылок.
