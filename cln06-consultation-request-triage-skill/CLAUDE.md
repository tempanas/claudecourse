# Consultation Request Triage Skill

## Роль агента

Агент анализирует обезличенные заявки студентов на консультацию, классифицирует тариф и предварительный тип консультации, выявляет недостающие данные и риски, формирует следующий внутренний шаг и сохраняет структурированный отчёт в `reports/`.

Зерокодер19 - это название должности сотрудника, который организует консультации, а не имя человека.

Проект предназначен для тестирования проектного skill `consultation-request-triage` в рамках домашнего задания CLs03, урок 6 «Скиллы в Claude Code».

## Правило нейминга

- Название skill: `consultation-request-triage`.
- Файл инструкции skill: `.claude/skills/consultation-request-triage/skill.md`.
- Основной результат тестирования: `reports/2026-06-consultation-request-triage-result.md`.
- Новые рабочие файлы именовать строчными латинскими буквами в kebab-case, если заданием не указано другое.

## Правила безопасности

- Работать только внутри текущей проектной папки.
- Не удалять и не переименовывать существующие файлы.
- Не отправлять сообщения студентам, кураторам, экспертам, VIP-менеджерам или в рабочие чаты.
- Не создавать реальные Zoom-встречи, не открывать платёжные страницы и не изменять рабочие таблицы.
- Не выполнять внешние действия от имени Зерокодер19.
- Использовать только локальный анализ текста и локальное сохранение отчёта.
- Не использовать реальные ФИО, email, телефоны, Telegram-ссылки, Zoom-ссылки, ссылки на таблицы, профили студентов или оплату.
- Все примеры, идентификаторы и ссылки должны быть обезличенными и демонстрационными.
- Если во входных данных встретились чувствительные сведения, не копировать их в результат и заменить безопасным маркером.

## Источники

PDF-регламент использовать только как локальный источник бизнес-логики. В создаваемые документы переносить содержание в виде краткой обезличенной сводки без реальных ссылок и контактов.

Исходный PDF и временные материалы из `tmp/` могут содержать фрагменты внутреннего регламента. Не включать их в публичный репозиторий, архив домашнего задания или демонстрационные материалы. Для сдачи использовать только обезличенные Markdown-файлы проекта.
