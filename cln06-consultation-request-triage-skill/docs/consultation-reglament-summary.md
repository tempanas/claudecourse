# Краткая сводка регламента консультаций

Документ содержит обезличенную рабочую логику. Реальные контакты, ссылки, цены и платёжные реквизиты намеренно не приводятся.

## Приём запроса

Зерокодер19 получает запрос на организацию консультации и проверяет, достаточно ли информации для дальнейшего разбора. В проекте обрабатывается только текст заявки, без отправки ответов и внешних действий.

## Анализ профиля

Для стандартной заявки нужно определить курс, урок, тариф, поток и ответственного куратора, а также проверить наличие списка вопросов. Отсутствующие сведения фиксируются как пробелы для последующего уточнения.

## Проверка тарифа

- Базовый / Самостоятельный: консультации считаются платными; студент может выбрать куратора или эксперта.
- Бизнес / С поддержкой: в отдельных курсах могут быть предусмотрены бесплатные консультации. Нужно проверить условия курса, доступный лимит и соответствие вопросов учебному процессу.
- VIP / VIP+ / LUXURY / Корпоративный: действует отдельная логика через VIP-менеджера.

## Проверка вопросов

Список вопросов обязателен для подготовки стандартной консультации. Куратор оценивает, относятся ли вопросы к учебному процессу и может ли консультация быть бесплатной. Частичный или слишком общий список считается риском и требует уточнения.

## Бесплатная или платная консультация

Для Базового / Самостоятельного предварительный тип - платная. Для Бизнеса / С поддержкой бесплатность нельзя определять только по названию тарифа: нужны подтверждение условий курса, остатка консультаций и оценка вопросов. При недостатке данных результат - «требуется уточнение».

## Слот

Нужно получить доступные слоты специалиста, предложить их студенту и зафиксировать согласованное время. В рамках этого тестового проекта skill только отмечает, указан и согласован ли слот.

## Оплата

Для платной консультации перед подготовкой ссылки на встречу требуется внутреннее подтверждение оплаты. Skill не проверяет реальные платежи и не открывает внешние системы, а только выставляет признак необходимости проверки.

## Zoom

После согласования слота и, если применимо, подтверждения оплаты подготавливается ссылка на встречу. Skill не создаёт реальную Zoom-ссылку, а фиксирует необходимость действия или использует демонстрационный плейсхолдер.

## Фиксация в таблице

Стандартные консультации по тарифам Базовый / Самостоятельный и Бизнес / С поддержкой должны быть отражены в рабочей таблице. Skill не изменяет таблицу, а отмечает необходимость фиксации.

## Отдельная логика VIP

Для VIP, VIP+, LUXURY и Корпоративного тарифа запрос на Zoom поступает от VIP-менеджера. Анализ профиля студента и подготовку консультации выполняет VIP-направление. Задача Зерокодер19 ограничена созданием ссылки на встречу, а фиксацию консультации выполняет VIP-менеджер. В тестовом проекте реальная ссылка не создаётся: skill только формирует внутренний следующий шаг.
