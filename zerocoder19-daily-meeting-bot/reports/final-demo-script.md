# Сценарий защиты на 3-5 минут

## 1. Что это за проект

Здравствуйте. Мой финальный проект называется **Zerocoder19 Daily Meeting Bot**.
Это Telegram-бот для контроля встреч на текущий и следующий день.

Бот показывает расписание и сразу отмечает, есть ли в событии Zoom-ссылка.

## 2. Какую проблему решает

В работе Зерокодер19 нужно контролировать консультации, Q&A, Zoom-встречи и
другие живые созвоны. Главная боль - быстро понять, какие встречи сегодня
запланированы и какие события не готовы, потому что в них нет ссылки для
подключения.

Бот помогает не просматривать календарь вручную. Он делает короткую сводку в
Telegram и подсвечивает события, которые нужно проверить.

## 3. Как запустить

Локальный запуск выполняется из корня проекта:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python3 -m src.bot
```

Для безопасной демонстрации можно оставить:

```dotenv
USE_DEMO_MODE=true
```

Тогда бот будет читать обезличенные события из `data/sample_events.json`.

## 4. Как показать `/today`

В Telegram нужно открыть личный тестовый чат с ботом и отправить:

```text
/today
```

В демо-режиме бот покажет встречи на текущий день:

- консультацию тарифа Бизнес с найденной Zoom-ссылкой;
- Q&A по потоку с найденной Zoom-ссылкой;
- встречу без Zoom-ссылки, которую нужно проверить в календаре.

После этого можно отправить:

```text
/tomorrow
```

Бот покажет события на завтра и также отметит, где Zoom найден, а где нет.

## 6. Что планируется развить дальше

Дальше проект можно развивать в сторону полноценного рабочего помощника:

- добавить автоматические напоминания о встречах без Zoom;
- добавить ежедневную утреннюю сводку;
- добавить фильтры по типам встреч;
- добавить тесты;
- улучшить обработку ошибок Google Calendar API;
- подготовить деплой на сервер или облачную платформу.

Итог: MVP уже решает основную задачу - показывает расписание и быстро находит
события, которые требуют проверки перед созвоном.
