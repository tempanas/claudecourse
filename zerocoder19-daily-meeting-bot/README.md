# Zerocoder19 Daily Meeting Bot

Telegram-бот для ежедневного контроля живых встреч, консультаций, Q&A и
Zoom-созвонов из календаря.

> Зерокодер19 - название должности, а не имя человека.

## Проблема

В рабочем расписании важно быстро понять:

- какие встречи запланированы на сегодня и завтра;
- во сколько начинается каждый созвон;
- добавлена ли в событие Zoom-ссылка;
- какие события нужно проверить вручную до начала встречи.

Ручная проверка календаря занимает время и повышает риск пропустить событие без
ссылки на подключение.

## Пользователь

Основной пользователь MVP - специалист в должности Зерокодер19, который
контролирует своевременность консультаций, Q&A, Zoom-встреч и других живых
созвонов.

## Возможности MVP

- показывает встречи на сегодня;
- показывает встречи на завтра;
- работает с обезличенным JSON или Google Calendar API;
- ищет ссылки `zoom.us` в описании, месте и данных видеоконференции;
- отмечает события со ссылкой статусом `Zoom: найден ✅`;
- помечает события без ссылки статусом `Zoom: не найден ⚠️`;
- подсказывает проверить проблемное событие в календаре;
- работает в стабильном демо-режиме без Google API и секретных ключей.

## Команды

| Команда | Назначение |
| --- | --- |
| `/start` | Приветствие и список команд |
| `/help` | Справка |
| `/health` | Статус бота, режим и часовой пояс |
| `/demo` | Описание демо-режима и встречи на сегодня |
| `/today` | Расписание на сегодня |
| `/tomorrow` | Расписание на завтра |

## Локальный запуск

Требуется Python 3.9 или новее.

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

Создайте тестового Telegram-бота через BotFather и добавьте токен только в
локальный файл `.env`:

```dotenv
TELEGRAM_BOT_TOKEN=replace_with_test_bot_token
GOOGLE_CALENDAR_ID=primary
TIMEZONE=Europe/Minsk
USE_DEMO_MODE=true
```

Запуск:

```bash
python -m src.bot
```

Не используйте для проверки реальные рабочие чаты. Откройте личный диалог с
тестовым ботом и выполните `/demo`, `/today` и `/tomorrow`.

## Демо-режим

Демо включён по умолчанию через `USE_DEMO_MODE=true`. Сервис читает
`data/sample_events.json`. Поле `day_offset` делает данные многоразовыми:

- `0` означает текущий день;
- `1` означает следующий день.

Поэтому демонстрация работает независимо от даты запуска. Все участники
обезличены (`student_001`, `curator_a`, `stream_demo`), а Zoom-ссылки содержат
фиктивные идентификаторы и предназначены только для проверки парсера.

## Подключение Google Calendar

Интеграция использует OAuth 2.0 только для чтения календаря.

### 1. Создать и настроить проект Google Cloud

1. Откройте [Google Cloud Console](https://console.cloud.google.com/).
2. Создайте отдельный проект или выберите существующий тестовый проект.
3. Откройте **APIs & Services > Library**.
4. Найдите **Google Calendar API** и нажмите **Enable**.
5. Настройте экран согласия в **Google Auth platform**.
6. Для внешнего приложения добавьте свой Google-аккаунт в тестовые
   пользователи.

Официальная инструкция:
[Google Calendar Python quickstart](https://developers.google.com/workspace/calendar/api/quickstart/python).

### 2. Скачать `credentials.json`

1. Откройте **Google Auth platform > Clients**.
2. Нажмите **Create Client**.
3. Выберите тип приложения **Desktop app**.
4. Создайте OAuth client и скачайте JSON-файл.
5. Переименуйте скачанный файл в `credentials.json`.
6. Поместите его в корень проекта рядом с `README.md`.

### 3. Переключить режим

В локальном `.env` установите:

```dotenv
GOOGLE_CALENDAR_ID=primary
TIMEZONE=Europe/Minsk
USE_DEMO_MODE=false
```

`GOOGLE_CALENDAR_ID=primary` использует основной календарь авторизованного
аккаунта. Для другого календаря укажите его идентификатор из настроек Google
Calendar.

Запустите бота:

```bash
python3 -m src.bot
```

При первом запросе `/today` или `/tomorrow` откроется браузер для OAuth
авторизации. После успешного входа приложение создаст локальный `token.json`.
Следующие запросы используют и при необходимости обновляют этот токен.

Если `credentials.json` отсутствует или авторизация не настроена, бот ответит:

```text
credentials.json не найден в корне проекта. Скачайте OAuth Client ID типа Desktop app из Google Cloud Console и положите файл в корень проекта.
```

Сервис запрашивает события между началом и концом выбранного дня в часовом
поясе `TIMEZONE`. Он читает `summary`, `start`, `end`, `location`,
`description`, `hangoutLink` и `conferenceData.entryPoints`, а затем приводит
события к тому же формату, который использует демо-сервис.

## Секреты и безопасность

Нельзя публиковать или коммитить:

- `.env` и `.env.local`;
- `credentials.json`;
- `token.json`;
- реальные Telegram-токены;
- реальные Zoom-ссылки;
- ФИО, телефоны и email участников.

`credentials.json` содержит OAuth client ID и client secret приложения.
`token.json` содержит пользовательские access/refresh tokens и может давать
доступ к календарю в пределах выданных разрешений. Оба файла являются
секретными и не должны попадать в Git, сообщения или скриншоты.

Эти файлы уже добавлены в `.gitignore`. Перед публикацией дополнительно
проверьте `git status` и историю Git. Если секрет уже был закоммичен, одного
добавления в `.gitignore` недостаточно: credential или token нужно отозвать и
создать заново.
