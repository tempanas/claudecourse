# Заметка к домашнему заданию CLb01

## Что демонстрируется в проекте

Проект показывает автоматизацию первичного анализа студенческих претензий. Python-скрипт читает обращения из CSV, передаёт каждую строку локальной модели и сохраняет структурированный результат: тональность, тип обращения, приоритет и рекомендуемое действие.

## Почему данные обезличены

Даже учебный пример должен учитывать требования конфиденциальности. Поэтому в наборе нет реальных ФИО, телефонов, email, ссылок, платёжных данных и других идентификаторов. Используются вымышленные псевдонимы вида `Студент-01` и искусственно созданные тексты обращений.

## Где используется локальная модель

Модель запускается через Ollama на компьютере пользователя. Скрипт отправляет запросы только на локальный адрес `http://localhost:11434/api/generate`. Облачные API и внешние сервисы в процессе анализа не используются.

## Польза для работы Зерокодер19

Зерокодер19 — название должности. Для этой роли решение помогает быстрее выполнять первичную сортировку потока обращений:

- находить запросы на возврат;
- выделять негативные и срочные претензии;
- отделять технические проблемы от вопросов по курсу;
- определять обращения, которым нужна консультация;
- предлагать следующее рабочее действие.

Модель не заменяет решение сотрудника претензионного сектора. Она подготавливает структурированную первичную классификацию, чтобы специалист мог быстрее расставить приоритеты и проверить обращение.
