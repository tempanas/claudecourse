# Локальный анализ претензий студентов

Проект для продвинутого варианта домашнего задания CLb01 «Локальные модели. Автоматизация рутинных задач». Он демонстрирует первичную классификацию обезличенных обращений студентов с помощью локальной модели Ollama.

«Зерокодер19» в контексте проекта — название должности, а не имя человека.

## Почему используется локальная модель

Претензии могут содержать чувствительный рабочий контекст. Локальная модель обрабатывает файлы на компьютере и не требует отправки текста обращений во внешние сервисы. В демонстрационном наборе используются только вымышленные псевдонимы и искусственные сообщения без ФИО, телефонов, email, ссылок и платёжных данных.

## Структура проекта

```text
data/       исходный и итоговый CSV
prompts/    инструкция для локальной модели
scripts/    Python-скрипт анализа
reports/    шаблон отчёта
```

## Требования

- Python 3.9 или новее;
- установленная Ollama;
- локально скачанная модель;
- запущенный API Ollama по адресу `http://localhost:11434`.

Скрипт использует только стандартную библиотеку Python. Устанавливать дополнительные Python-пакеты не нужно.

## Запуск Ollama

Установите Ollama с официального дистрибутива для вашей операционной системы. После установки запустите приложение Ollama или сервер из терминала:

```bash
ollama serve
```

Если Ollama уже работает как фоновое приложение, повторно запускать `ollama serve` не требуется.

## Скачивание модели

Например, можно скачать модель:

```bash
ollama pull llama3.2:3b
```

Проверить список доступных локальных моделей:

```bash
ollama list
```

## Пользовательская модель с контекстом 32K

В проекте есть `Modelfile.claims-32k`, созданный на основе `llama3.2:3b`. В нём задан параметр:

```text
PARAMETER num_ctx 32768
```

Создать пользовательскую модель:

```bash
ollama create claims-llama-32k -f Modelfile.claims-32k
```

## Запуск анализа

Из корня проекта выполните тест на первых пяти строках:

```bash
python3 scripts/analyze_student_claims.py --model claims-llama-32k --limit 5
```

Для обработки всех строк:

```bash
python3 scripts/analyze_student_claims.py --model claims-llama-32k
```

При необходимости таймаут одного запроса можно увеличить:

```bash
python3 scripts/analyze_student_claims.py --model claims-llama-32k --timeout 240
```

Если пользовательская модель не запускается, используйте базовую:

```bash
python3 scripts/analyze_student_claims.py --model llama3.2:3b
```

## Результат

Исходные данные находятся в:

`data/2026-06-student-claims-demo-data.csv`

После успешного запуска создаётся:

`data/2026-06-student-claims-demo-data-analyzed.csv`

В итоговый CSV добавляются колонки:

- `sentiment`;
- `claim_type`;
- `priority`;
- `action`.

Если Ollama не запущена или модель недоступна, скрипт завершится с понятным сообщением. Если отдельный ответ модели не соответствует допустимому формату, для этой строки записываются fallback-значения, выводится предупреждение и обработка продолжается. При ошибке подключения уже существующий итоговый файл не перезаписывается.

Отчёт по тестовому запуску находится в:

`reports/2026-06-local-claims-analysis-report.md`

## Конфиденциальность

Скрипт обращается только к локальному API `localhost`. Для рабочих данных перед использованием необходимо отдельно убедиться, что обращения обезличены и не содержат персональных, контактных, ссылочных или платёжных данных.
